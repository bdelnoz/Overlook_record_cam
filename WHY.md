# Why this repository exists

## Problem addressed

Operational review workflows need long-running, segmented desktop recordings with reliable audio capture and minimal manual intervention.

## Design goals

- Keep recording active for long sessions.
- Prevent oversized files with segmented output.
- Preserve evidence context with timestamped naming.
- Offer optional movement highlighting for rapid review.
- Provide restore-friendly delete workflow instead of irreversible cleanup.

## Practical outcomes

- Faster post-incident investigation.
- Better traceability with logs and generated file summaries.
- Flexible operation (video only, diff only, or both).
